  <action>
    The current seed loop calls `syncResourceToMeilisearch(resource)` but discards
    the returned `EnqueuedTask`. Fix by collecting task UIDs and calling
    `meilisearchClient.waitForTasks()` before `process.exit(0)`.

    Import `meilisearchClient` at the top of the file (it is already exported from
    `../lib/meilisearch`):

    Change the import line from:
    ```typescript
    import { syncResourceToMeilisearch } from '../lib/meilisearch';
    ```
    To:
    ```typescript
    import { meilisearchClient, syncResourceToMeilisearch } from '../lib/meilisearch';
    ```

    Replace the Meilisearch sync loop (lines 294-297 approximately):
    ```typescript
    // Before:
    for (const resource of insertedResources) {
      await syncResourceToMeilisearch(resource);
    }
    console.log('Synced resources to Meilisearch');
    ```

    ```typescript
    // After:
    const taskUids: number[] = [];
    for (const resource of insertedResources) {
      const task = await syncResourceToMeilisearch(resource);
      taskUids.push(task.taskUid);
    }

    console.log(`Waiting for ${taskUids.length} Meilisearch indexing tasks...`);
    await meilisearchClient.waitForTasks(taskUids);
    console.log('Synced resources to Meilisearch');
    ```

    Do not change any other part of the file. The `finally(() => process.exit(0))`
    at the bottom already ensures the process exits after `seed()` resolves — the
    `await` above guarantees Meilisearch tasks complete before that happens.

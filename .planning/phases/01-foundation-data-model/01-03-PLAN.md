Create docker-compose.yml in project root with Meilisearch service configuration:

```yaml
services:
  meilisearch:
    image: getmeili/meilisearch:v1.16
    container_name: quantumthreat-meilisearch
    ports:
      - "7700:7700"
    environment:
      - MEILI_MASTER_KEY=${MEILISEARCH_MASTER_KEY}
      - MEILI_ENV=development
    volumes:
      - ./data.ms:/meili_data
    restart: unless-stopped
```

**Configuration explanation:**
- Image: getmeili/meilisearch:v1.16 (latest stable as of April 2026)
- Port: 7700 (default Meilisearch HTTP port)
- MEILI_MASTER_KEY: Read from environment, required for authentication
- MEILI_ENV: development mode enables admin UI at http://localhost:7700
- Volume: ./data.ms persists index data across container restarts
- Restart policy: unless-stopped ensures service survives reboots

**Implementation requirements:**

**Step 1: Client initialization with error handling**
```typescript
import { MeiliSearch } from 'meilisearch';

if (!process.env.MEILISEARCH_URL || !process.env.MEILISEARCH_MASTER_KEY) {
  throw new Error('Missing Meilisearch environment variables');
}

export const meilisearchClient = new MeiliSearch({
  host: process.env.MEILISEARCH_URL,
  apiKey: process.env.MEILISEARCH_MASTER_KEY,
});
```

**Step 2: Index configuration function**

Create `initializeMeilisearchIndex()` async function that:
- Gets the 'resources' index
- Calls updateSettings() with these exact settings:
  - searchableAttributes: ['title', 'summary', 'authors', 'tags'] (title ranked first per RESEARCH.md)
  - filterableAttributes: ['type', 'date', 'technical_level', 'tags']
  - sortableAttributes: ['date']
  - rankingRules: ['words', 'typo', 'proximity', 'attribute', 'sort', 'exactness']
  - typoTolerance: { enabled: true, minWordSizeForTypos: { oneTypo: 5, twoTypos: 9 } }
- Logs 'Meilisearch index configured' on success

**Step 3: Sync functions for future Server Actions**

Create `syncResourceToMeilisearch(resource)` function that:
- Takes resource parameter with fields: id, title, summary, type, date, authors, tags, technical_level, external_link
- Converts date to ISO string (Meilisearch requires ISO format, not Date objects)
- Calls index.addDocuments([document])
- Returns the task (for async tracking if needed)

Create `deleteResourceFromMeilisearch(resourceId)` function that:
- Takes resourceId (string)
- Calls index.deleteDocument(resourceId)
- Returns the task

**Type safety:** Import Resource type from db/schema for syncResourceToMeilisearch parameter type.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Application → Meilisearch | Search queries and document sync operations |
| Docker → Host | Meilisearch container exposes port 7700 to localhost |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-08 | Information Disclosure | MEILISEARCH_MASTER_KEY | mitigate | Store master key in .env (gitignored), never hardcode in source files |
| T-01-09 | Elevation of Privilege | Meilisearch admin UI | accept | Development environment only, port 7700 not exposed in production (use MEILI_ENV=production + private network) |
| T-01-10 | Denial of Service | Meilisearch port 7700 | accept | Localhost binding only in dev, production will use internal network or Meilisearch Cloud |
| T-01-11 | Tampering | Meilisearch index data | mitigate | PostgreSQL is source of truth, Meilisearch can be rebuilt from database via sync functions |
</threat_model>
